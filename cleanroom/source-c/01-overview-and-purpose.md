# Source C — overview and purpose

## Mission

Source C provides **lightweight IP address management** for teams that need working subnet and address workflows without operating a large multi-service Python platform. Public README emphasizes **simplicity** and **Ajax-driven** interaction.

## Typical users

- SMB and mid-market IT teams.
- Operators who prioritize **fast adoption** on **LAMP** stacks over exhaustive network digital twins.

## Positioning vs Source A

Source C is **IP-centric tooling** with adjacent features (devices, racks, light DDI views). Source A is a **network source of truth and automation platform** with deep DCIM/IPAM graphs, jobs, webhooks, and Git-backed content. Use Source C for **minimal footprint**; use Source A for **automation hub** requirements.
