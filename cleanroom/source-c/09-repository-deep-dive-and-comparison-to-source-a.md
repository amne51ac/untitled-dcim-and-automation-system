# Source C — repository deep dive and comparison to Source A

## Scale and shape

Hundreds of PHP modules; heavy **`tools`** expansion—shows **breadth of operator utilities** over **deep unified graph** rigor.

## Architectural summary

**LAMP monolith** optimized for **IPAM-first** teams. **Not** a programmable multi-tenant network automation core like Source A.

## Comparison table

| Dimension | Source C | Source A |
|-----------|----------|----------|
| **Runtime** | PHP sections | Django + Celery workers |
| **Model depth** | IPAM + adjacent tools | Full DCIM/IPAM/circuits/cloud/VPN/LB/wireless + extras |
| **Automation** | API scripts | Jobs, hooks, Git, webhooks, event brokers |
| **Hosting cost** | Often lower | Higher (workers, cache, DB tuning) |

## Greenfield takeaway

Borrow Source C ideas for **simplicity** and **tool sprawl UX** where appropriate; borrow Source A for **platform extensibility** and **network automation** integration.
