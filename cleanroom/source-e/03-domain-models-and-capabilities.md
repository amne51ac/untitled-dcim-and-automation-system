# Source E — domain models and capabilities

## IP management

**IPv4/IPv6** hierarchical plans, subnets, ranges, addresses—with **delegation** between organizations for MSP scenarios.

## Network documentation

WAN links, VLANs, devices, interfaces, VRFs, AS numbers—network documentation adjacent to pure IPAM.

## CMDB linkage

IPs attach to **configuration items**—applications, servers, interfaces—creating **service-to-address** traceability.

## DNS (zone extension)

Authoritative **zone** and **record** modeling aligned with IP space—core **DDI** story when both extensions are installed.

## Discovery

Additional Git repositories in the same organization provide **discovery probes** and engines—not required for core documentation but part of the product ecosystem.
