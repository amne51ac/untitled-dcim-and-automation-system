# Source G — APIs and integration

## Integration style

**Script-centric PHP** with SQL—automation typically via **direct database** or **custom wrappers** rather than a unified GraphQL/REST platform like Source A.

## VMware

**poll_esx_inventory** bridges hypervisor inventory into the DCIM dataset—useful for **physical/virtual** reconciliation.

## Contrast

Source A offers **first-class REST + GraphQL + webhooks**. Source G prioritizes **UI reports** and **operator scripts**.
