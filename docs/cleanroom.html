Clean-room research

The cleanroom/ tree holds capability and architecture notes derived from reference platforms (open-source sources A–H and a neutral commercial marketing survey, Additional Source 1). It informs requirements and differentiation; this codebase does not copy third-party source.

How we use it

  1. Requirements & domain model — Consolidate DCIM, IPAM, circuits, cloud-adjacent objects, jobs, events into one coherent provider-oriented model.
  2. Platform vs. product — Core platform (auth, tenancy, audit, APIs, jobs, plugin host) vs. optional apps (compliance packs, discovery, reporting).
  3. Non-functional targets — SLOs for API latency, ingestion, workers, RPO/RTO, horizontal scaling.
  4. Differentiation — Borrow ideas, not code; document analytical limitations explicitly.
